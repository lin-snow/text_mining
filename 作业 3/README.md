# 作业 3：基于降维与近似最近邻的中文科技资讯推荐系统

> 文本信息挖掘概论 · 作业 3　|　林奕宏 3123004449 软件工程 2 班

题面：**以 LSA/LSI/LDA 等降维为出发点，设计一套基于降维技术的推荐系统框架**。
本作业把 221 篇中文科技 RSS 文章映射到低维语义空间（LSA 截断 SVD / LDA 主题分布），
在低维空间用最近邻检索做"读了这篇 → 还想读这些"的推荐，并以 PCA/t-SNE/UMAP 可视化、
以一种自创的"新词推荐锚定力（NAP）"机制评估作业 2 新词对推荐的有效性。

## 核心结论

- **以推荐质量（而非方差能量）选降维维度**：在主题命中率与"对全维 TF-IDF 的邻域保持"两项指标上选定 **k\*=80**（1897→80 维，约 24 倍压缩，保住 80.7% 全维近邻结构）；
- **推荐有效**：LSI 推荐的主题一致命中率 0.286，是随机基线（0.118）的 **2.4 倍**；耳机文章召回耳机、Gemma 4 跨 4 家媒体召回同模型报道（cos 0.68–0.97）；
- **近似最近邻近乎无损**：近似召回相对精确召回 **recall@10 = 0.995**；
- **可视化**：UMAP 投影 trustworthiness 最高（0.75）> t-SNE（0.73）> PCA（0.68）；
- **脑洞新词评估 NAP**：作业 2 的 80 个新词 NAP 全为正；移除最强锚点 *千问* 致其文档簇内相似度骤降 **23.4%**；NAP 与作业 2 词频得分仅弱负相关（r=−0.20），二者互补。

## 实现说明

题面 [2] 推荐的 Annoy 库（1.17.3）在本机 Python 3.13 下 `get_nns_by_item` 仅返回单结果并段错误（ABI 不兼容，源码重编译亦然），故近似最近邻改用**同族、且为 UMAP 召回引擎**的 `pynndescent`（NN-Descent），思想一致、结论可迁移，已在代码与报告中如实标注。
